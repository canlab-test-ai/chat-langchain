1. All-inclusive No-code SaaS (Software-as-a-Service)

2. Headless Service

3. canD API Service

There are three ways to leverage canD:

1. All-inclusive No-code SaaS (Software-as-a-Service)
Without the need for developers, you can launch a service that includes all desired features using canD's all-inclusive SaaS. This includes web services as well as native apps (iOS/Android). With this approach, companies that find it challenging to maintain software development departments can launch competitive Web2/Web3 businesses and integrated online-offline platforms with minimal or no developers.

3. canD API Service
If you already have an existing service in operation, you can maintain your existing service while using the core features of canD modules by calling API functions. Some canD APIs are also provided in the form of Discord plugins, making it easy to utilize canD's features within Discord communities."

CANpass is conceptually and philosophically different from the single sign-on systems of traditional major corporations. Traditional corporations collect and own user data extensively, providing it to services that adopt their login systems. In the canD ecosystem, CANpass is not a centralized entity that collects and owns user data; instead, individual services own and manage user data.
